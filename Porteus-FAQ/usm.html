WHAT ARE MODULES?

Porteus is modular by design. In other operating systems where you might download and install a program, in Porteus you simply 'activate a module'. Activating a module simply injects the directory structure of the module into the live file system after which it is ready to use. If you don't need it any more you simply 'deactivate' the module.

WHAT ARE DEPENDENCIES AND WHY SHOULD I CARE?

Usually a linux application requires certain files (libraries) to be present in order to successfully run an application. An example would be that you want to start the application called 'codeblocks' but you find that it requires another application called wxwidgets before it can start. Anyone that has attempted to compile their own packages in Linux has no doubt experienced 'dependency hell'. This is where you download wxwidgets only to find that IT requires two other packages to run. It may be the case that these packages required by wxwidgets also require other packages ... and this can go on until you give up following the chain, or find that finally no other packages are required. Nobody likes hunting around for required packges and this is one reason that a good package manager resolves these dependencies for the user.

WHAT ARE THE FEATURES OF USM?

USM is a hybrid CLI/GUI program which means you can run it from a console (type: usm) or with a graphical user interface from the user application menu. It is an application written specifically for Porteus; but designed with slackware users in mind. In the slackware world it is known as USM (Unified Slackware Package Manager).

Primarily, USM was designed for downloading and converting slackware packages from multiple repositories, but it also has many other functions:

  • Package tools
  • If a dependency is missing, USM will download it
  • Missing system dependency resolution
  • Conversion of Slackware packages to Porteus modules
  • Various Porteus module & Slackware package manipulation tools
  • Checking for updates to the database and program

Features are always being changed and added to USM as and when they are requested. If you have a request please post it on the Porteus forum in the appropriate thread.

WHERE ARE MY CONFIG FILES?

USM stores a couple of config files. /etc/usm/ holds the configuration files which can also be edited using the GUI

HOW IS A PORTEUS MODULE DIFFERENT FROM A SLACKWARE PACKAGE?

It is not really all that different at all. A Porteus moule does not require installing (see WHAT IS A PORTEUS MODULE?) in the way that a slackware package does. The file structure of a Porteus module is very similar to a slackare package with the addition of an extra file, and the removal of unneeded files. A list of 'required libraries' & 'offered libraries' is put into /var/porteus/package.info. The /var/lib/pkgtools/packages file is also modified to include a CATEGORY: line and the LOCATION: line is changed to suit the Porteus server.

CAN I SUBMIT A MODULE I CREATED?

Modules that you create can be shared in the Modules section of the forum, by uploading to a filesharing site and posting the link. Shared Modules should be made in an "Always Fresh" environment to avoid missing dependencies. Please specify which Desktop(s) is targeted, supply a md5sum and a list of the packages included.

WHAT IS A PORTEUS COMPLIANT MODULE?

One that works :p .. Also not illegal or pirated software.

I FOUND A BUG IN USM, HOW CAN I NOTIFY THE DEVELOPER?

Start a thread in the bug report topic on the Porteus forum with a description of your find.
